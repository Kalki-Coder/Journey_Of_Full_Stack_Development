this is simple, beginner-friendly summary of essential Git and GitHub concepts and commands.
It is designed for complete beginners who are learning programming and version control together.
        What is Git?
Git is a version control system that runs on your computer. It tracks changes in your code, helps you go back to previous versions, and prevents loss of work.
        What is GitHub?
GitHub is an online platform that stores Git repositories. It is used to back up code, share projects, and collaborate with others.
        Basic Git Workflow
Write code → Add changes → Commit changes → Push to GitHub
        One-Time Setup Commands
• git --version : Check if Git is installed 
• git config --global user.name "Your Name" : Set your name 
• git config --global user.email "email@example.com" : Set your email
        Starting a Project
• git init : Start Git tracking in a folder 
• git status : Check file status (most used command)
        Saving Changes
• git add . : Add all changed files to staging area
• git commit -m "message" : Save a snapshot of your code
        Viewing History & Changes
• git log : View commit history 
• git diff : See exact changes in files
        Undoing Mistakes
• git restore filename : Undo uncommitted changes
• git reset filename : Remove file from staging area
        Branches
• git branch : List branches 
• git branch name : Create new branch 
• git switch name : Switch branch
        GitHub Commands
• git remote add origin URL : Connect local repo to GitHub 
• git push origin main : Upload code to GitHub 
• git pull origin main : Download latest changes 
• git clone URL : Copy a repository from GitHub
        Important File
.gitignore : Lists files and folders Git should not track (passwords, cache files, etc.)
        Daily Commands to Remember
• git status 
• git add . 
• git commit -m "message" 
• git push
